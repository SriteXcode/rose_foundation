# Security Policy

## Supported Versions

We actively monitor and fix security vulnerabilities for the following versions of the Black Rose Foundation platform:

| Version | Supported |
| ------- | --------- |
| 1.0.x   | ✅ Yes    |
| < 1.0.0 | ❌ No     |

---

## Reporting a Vulnerability

We take the security of the Black Rose Foundation NGO Management System seriously. If you believe you have found a security vulnerability in this application, please do **not** report it via public issues or discussions.

Instead, please report security vulnerabilities directly to our maintainers:

*   **Email**: [blackrosefoundation111@gmail.com](mailto:blackrosefoundation111@gmail.com)

### What to Include in a Report
To help us triage and resolve the issue quickly, please provide the following details:
1.  **Type of issue** (e.g., SQL Injection, XSS, CSRF, remote code execution).
2.  **Location/API path** of the vulnerable endpoint.
3.  **Step-by-step instructions** or a proof-of-concept (PoC) script to reproduce the issue.
4.  **Impact** of the vulnerability (what could an attacker achieve?).

---

## Our Response Process

1.  **Acknowledgment**: We will acknowledge receipt of your vulnerability report within 48 hours.
2.  **Investigation**: We will investigate and attempt to reproduce the reported issue.
3.  **Remediation**: If confirmed, we will work on a patch to resolve the vulnerability.
4.  **Disclosure**: We will coordinate the release of the patch and advisory, prioritizing the security of live sites running the platform.
